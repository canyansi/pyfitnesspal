PyFitnessPal

a simple myfitnesspal clone made with django

Motivation

MyFitnessPal makes it very easy to enter data, with a very large database and multiple apps (web, iOS, Android). However, it's not so easy to get reports or data out of it. There's only one highcharts based line graph, and no click-to-export anywhere.

PyFitnessPal doesn't have the features for input or sharing of data with other users, but is planned to be more extensible in the data importing, reporting and exporting departments.

Deploy on Heroku

You can deploy directly to Heroku with the button below.

The one-click deploy will automatically:

  • Initialize the database
  • Create a single user specified by you

Manual Setps

After deploying, run the Django migrate command to initialize your postgres database:

heroku clone -a new-app-name
heroku run python manage.py migrate

Then, create your login credentials like this:

heroku run python manage.py create_user my_awesome_username supersecretpassword

Run the Django syncdb command to set up the root administrator account:

heroku run python manage.py syncdb

Once you've deployed, you can easily clone the app and make modifications:

vim templates/login.html
git add .
git commit -m "updating login view"
git push heroku master

Running Locally

Install Python 2.7.9 locally, and create and activate a virutalenv using the create_venv.sh shell script:

brew install python
pip install virtualenv
./create_venv.sh

You must use the twelve-factor approved method to specify your database connection url.

As an example, if you want the Django, vanilla sqlite setup, set the DATABASE_URL environment variable before doing anything with manage.py:

export DATABASE_URL=sqlite:///$(pwd)/db.sqlite3

an sqlite backend can also be run in memory:

export DATABASE_URL=sqlite://:memory:

Use honcho to start the application with gunicorn, pretty much like Heroku does:

cat>>.env<<-EOF
DATABASE_URL="sqlite:///$(pwd)/db.sqlite3"
SECRET_KEY="$(./.make_secret.py)"
PYFITNESSPAL_USERNAME="demo"
PYFITNESSPAL_PASSWORD="Passw0rd!"
EOF
./.venv/bin/python manage.py migrate
./.venv/bin/python manage.py create_user
.venv/bin/honcho start

Management

Set up an initial administrator account with the syncdb command:

./.venv/bin/python manage.py syncdb
